About

 Monday, 5 November 2018

Over the years, I have collected snippets of code which refused to run as advertised. Getting the code to compile–and that too on a Windows machine–was sometimes frustrating but mostly irritating. TBH, I am not a full-time developer; I have a regular day job. So, getting even small segments in the NLP pipeline to work often became a herculean task. And, once I would gt into a challenging spot, the Arian ram in me would refuse to give up untill I had this sorted out.

For instance, getting the StanfordNLP package https://stanfordnlp.github.io/CoreNLP/ to compile took me a good 18 hours. Even with the latest build and Java-10, the system would not respond to server pings. Thousand of runs to Stackoverflow later, I found a comment which said SatnfordNLP will only compile without hassles with Java8. At other times, the documentation and usage followed the *nix conventions, which meant that I had to spend some time looking at the internals to figure out the equivalent conventions in Windows.

Like, the Chef Gusteau said, “Anyone can cook,” I to believe open source code needs to be far more accessible than it currently is.

This is my humble way of giving back to the community.
